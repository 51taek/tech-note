문제가 생겼을 때 내부 엔진을 보기 위해서는 INFO 명령어를 이용하지만 지연이나 메모리 관련 문제가 생겼을 때는 INFO 만으로 해결하기 어려움

### 지연이나 메모리 관련 문제가 발생했을 때 봐야 하는 것

- 특정 명령어만 느려지거나 전반적으로 명령어 처리가 느려지는지 여부
- 특정 캐시 노드에서만 문제가 발생하는지 또는 모든 캐시 노드에서 문제가 발생하는지
- 특정 클라이언트에서만 문제가 발생하는지 또는 다른 클라이언트에서도 문제가 발생하는지
- 지속적으로 발생하는 문제인지 아니면 특정 시간에만 발생하는지 여부
- 문제의 발생 빈도

### 레디스 INFO로 알 수 있는 것들

- Server: 레디스 관련 일반 정보
    - 재시작 여부, 시작 후 경과된 시간, 적재된 설정 파일
- Clients: 현재 클라이언트 연결 현황
- Memory
    - 레디스가 사용할 수 있는 최대 메모리양을 주의깊게 봐야함
    - 메모리 단편화 문제가 발생할 수 있음
    - → mem_fragmentation 값이 1.5가 넘어가면 메모리 단편화 문제가 심한 것
    - 반대로 1.0 미만인 경우 스왑이 일어날 수 있음
    - user_memory_overhead
        - 레디스 서버에 할당된 내부 데이터 구조 관리를 위한 오버헤드 메모리양
- Persistence
    - RDB 스냅숏과 AOF 관련 정보 확인 가능
- Stats
    - 현재 연결 수와 시간당 연결 수를 보고 부하를 측정
    - 캐시 미스 & 캐시 히트 정보
- Replication
    - 레플리케이션 랙 확인 가능
- CPU
    - 레디스가 시작된 이후 누적된 CPU 시간
- Modules
- Commandstats: 레디스 명령어 통계
- Errorstats
- Latencystats: 지연 시간의 백분위 통계 분포(레디스 7.0 이상)
    - 여러 명령어의 p50/99,99.9 값 확인 가능
- Cluster
- Keyspace: 데이터베이스 관련 정보
    - 메모리가 부족한 상황에서 TTL이 제대로 걸려있는지 확인할 때 자주 사용

### 슬로우 로그

레디스 쿼리에서 실행시간이 긴 것들을 로그로 기록하는 기능

- slowlog-slower-than : 이 값보다 더 오래걸리면 로그로 기록
    - 지시자가 음수면 비활성화
- slowlog-max-len : 레디스 서버가 유지할 엔트리 수

### redis-cli 옵션

#### —latency

- 컨트롤 + C 로 종료할 때까지 샘플링 계속함
- 중간에 ping을 날리면서 클라이언트와 레디스 서버간의 지연 측정

#### —latency-history

- 네트워크 문제 원인 분석하는 데 좋음
- 시간이 지남에 따른 지연 시간 추이를 확인하는 데 좋음

#### —latency-dist

- 지연시간과 비율에 따라 색으로 구분된 스펙트럼 표시가 됨

#### —intrinsic-latency

- 시스템 고유 지연 시간을 지속적으로 측정
- 지연시간의 최대치가 갱신될 때마다 업데이트
- 실행중인 프로세스를 커널이 얼마나 오랫동안 지연시키고 있는지 측정

### 지연시간 모니터링

- LATENCY 명령어를 통해 사용할 수 있음
- 기본적으로 비활성화 되어있지만 사용하는 메모리양이 매우 적기 때문에 지연 시간 문제가 발생하기 전에는 활성화하는 것이 좋음
- LATENCY DOCTOR 명령어를 통해서 문제를 진단하고 관찰된 증상으로부터 어떤 조치를 취해야 하는지 사람이 확인하기 쉬운 형태로 알려줌
    - 지연 시간 스파이크 간의 기간, 그 사이에 발생한 스파이크 수, 평균 시간, 평균 절대 편차의 통계 데이터가 표시됨. 이벤트별로 추가 상세 정보가 표시될 수도 있음

### 소프트웨어 워치독

지정된 임계갑의 시간을 밀리 초 단위로 설정하며, 명령어 실행 시 실행 식나이 임계값을 초과하면, 레디스 서버 측에 EIP 레지스터와 백트레이스를 출력할 수 있는 기능

트러블 슈팅에 도움이 될 수 잇지만 아직까지는 실험적임

지연 시간 문제를 트러블 슈팅할 때 다른 디버그 수단으로는 문제를 해결할 수 없는 경우에 사용할 수 있는 최종 수단. 일시적으로 활성화

CONFIG SET 명령어로 watchdog-period 매개변수에 입력값으로 설정할 시간을 밀리초 단위로 지정

### 메모리 문제

MEOMORY 명령어를 통해서 메모리 사용량을 확인할 수 있음. (INFO보다 더 자세히)

MEMORY DOCTOR라는 진단기능이 잇음

MEMORY STATS로 메모리 세부 내용 확인 가능
